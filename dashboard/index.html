Module Map

One deployable Spring Boot application. Separate Maven modules own separate business capabilities. Communication stays explicit through application services and Spring events.

Runtime boundaries
Client HTTP clients REST consumers call the public API exposed by the executable app.
↓
Bootstrap ecommerce-app Single Spring Boot runtime, configuration, REST wiring and migrations.
Domain module catalog Products, stock rules, read projections and Redis-backed queries.
Domain module orders Order placement, idempotency, lifecycle and event publication.
Domain module payment Payment authorization simulation and event-driven processing.
1. API call Client places an order through ecommerce-app.
2. Stock orders calls catalog application service for reservation.
3. Event orders publishes OrderPlacedEvent.
4. Payment payment reacts through a Spring event listener.
Shared shared-kernel Small independent abstractions: domain events, publisher and shared errors.
Persistence PostgreSQL Schema is versioned through Flyway. Hibernate validates the model.
Cache Redis Used for read-side catalog caching, not as a fake integration bus.

How to Run Locally

Start PostgreSQL and Redis, then run the Spring Boot application from the executable module.

docker compose up -d
mvn -pl ecommerce-app -am spring-boot:run
mvn clean verify
